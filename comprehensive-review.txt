# Prompt

RESEARCH: Use multiple subagents to prepare to launch subagents to conduct comprehensive reviews on EVERY way you can conceive of. Divide into a few domains and a LOT of subdomains. Define what each subdomain should be looking out for.

<ITERATION LOOP>
RESEARCHER / IMPLEMENTOR / REVIEWER agents exist.
Assign 3 RESEARCHERs s.t. they will cover every subdomain. MAKE SURE the subagents are also actively looking for things to refactor / make better. As they return with results, review what they found and then send IMPLEMENTORs to implement / refactor / refine the suggested changes. When the IMPLEMENTORS return with results, send out the REVIEWER to review the work. This is a full cycle. Make sure the subagents are aware that other agents are working as well on the same codebase.

While this iteration loop is running, you will have 1 spare subagent to send out. Based on all the accumulated data you have, send this subagent as a RESEARCHER to try to find ANY additional subdomain paths that should be attempted to address.
When anything new has been found, push that to the stack of subdomains / tasks the 3 subagents are looping through.
</ITERATION LOOP>

Once finished, go through all changes and create a comprehensive report on what was done and why into @docs/long-review/. Make sure the filename is `{Bash(date +"%y%m%d-%H%M")}-{summary}.md`(Run the command).
