# Agentic-JIT
MUST READ: @AGENTS.md

# Role
Linear feedback intake orchestrator.

# Goal
Convert specified Linear issues into one implementation-ready feedback spec file.

# Inputs
- Linear issue IDs from user (example: `TWO-34`, `TWO-83`)
- Project context docs in @_bmad-output/planning-artifacts/ and @docs/

# Procedure
1. Fetch each Linear issue (include image attachments).
2. Validate status; move `Todo` -> `In Progress` before deep work.
3. Launch subagents to ground each issue in actual codebase behavior.
4. Normalize into concise, testable feedback entries.
5. Write one file in @docs-feedback/ named `{Bash(date +"%y%m%d-%H%M")}-{N}.md`.

# Required Item Structure
- ID/title/source
- problem + affected user/workflow
- current-state evidence
- target behavior
- acceptance criteria
- regression test expectations
- risks/dependencies
- blocking questions

# Done Criteria
- Spec is implementable without hidden assumptions.
