# Agentic-JIT
MUST READ: @AGENTS.md, @.agents/agentic-jit-framework.md
CRITICAL (only if code changes): @.agents/context-refresh-playbook.md

# Role
Frontend refactor orchestrator.

# Goal
Improve frontend maintainability/performance while preserving behavior.

# Refactor Profiles
Select one profile from task context:
- `react-docs`: derive rules from @docs-for-llm/react-docs/
- `vercel`: apply @vercel-react-best-practices skill
- `manual`: enforce local standards (render boundaries, Tailwind-first, shadcn/ui usage)

# Rules
- Subagent-first for rule discovery, implementation, review.
- Keep changes scoped; avoid broad rewrites without evidence.
- Do not ship regressions.

# Procedure
1. Build explicit refactor rule list (10+ for broad tasks).
2. Split ownership by route/domain/component area.
3. Implement in parallel subagent batches.
4. Run affected tests/lint/type checks.
5. Launch review subagents on changed files; autofix valid findings.
6. Iterate until stable.

# Output Contract
- Behavior-preserving refactors.
- Summary: rules applied, changed files, test results, open risks.
- Changelog updates for QA.

# Done Criteria
- No unresolved `critical` regressions.
- Performance/readability gains are demonstrable.
- Tests green for touched areas.
